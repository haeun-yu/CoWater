# Claude Development Process (상세 가이드)

> 기능을 구현할 때 단계별로 따를 프로세스

---

## 📋 전체 프로세스 흐름

> **규칙 확인**: CLAUDE.md의 "설계 → 승인 → 구현" 3단계를 상세히 설명합니다

```
【CLAUDE.md 핵심】
요청 → 설계 제시 → 사이드 이펙트 분석 → 사용자 승인 → 구현

【이 문서 상세】
  1️⃣ 요청 수신
  2️⃣ 이해 검증 (가정 명시)
  3️⃣ 설계 제시 (구현 전)
       ├─ 어떻게 만들 것인지
       ├─ 어느 파일을 수정할 것인지
       └─ 기술 선택 (JWT vs Session 등)
  4️⃣ 사이드 이펙트 분석
       ├─ 직접/간접 영향 파일
       ├─ DB, API, 설정 변경
       └─ 테스트, 배포 영향
  5️⃣ 사용자 승인 대기
       └─ GO 신호 없으면 구현 금지
  6️⃣ 계획 수립 (복잡하면 EnterPlanMode)
  7️⃣ 구현 & 검증
  8️⃣ 완료 & 기록
```

---

## 1️⃣ 요청 수신 & 이해 검증

```
사용자: "사용자 인증 기능 추가해줘"
  ↓
Claude 확인:
  - JWT? Session? 
  - 어느 파일들?
  - 뭐가 필요한가?
  → 물어보고 명확히 한 후 진행
```

> 상세는 CLAUDE.md의 "2️⃣ 이해 검증" 참고

---

## 2️⃣ 이해 검증 (가정 명시)

**모호한 부분은 구현 전에 명확히:**

```
가정하지 말아야 할 것:
- 기술 선택 (JWT vs Session, SQLite vs PostgreSQL)
- 영향 범위 ("아마 이 파일만 수정하면 될 것 같은데")
- 요구사항 범위 ("이것도 필요하겠지")

물어봐야 할 것:
- "이 기능의 성공 기준이 뭔가요?"
- "어떤 파일/서비스가 관련되나요?"
- "기존 동작을 유지해야 하나요?"
```

---

## 3️⃣ 설계 제시 (구현 전)

**승인 전에 모든 계획을 공개:**

```
【설계 안내】

▪ 방식: JWT 토큰 기반 인증
▪ 토큰 저장: localStorage (클라이언트)
▪ 갱신: 30분 만료, refresh token 사용

【파일 수정 예정】
- server/auth.py (새로 생성)
- server/routes.py (POST /login 추가)
- server/models.py (User 스키마 추가)
- client/auth.js (로그인 폼)
- database/migrations/ (schema 변경)

【사이드 이펙트】
- DB 마이그레이션 필수 (users 테이블)
- 기존 테스트 5개 수정 필요
- .env 파일에 JWT_SECRET 추가 필수
- 배포 과정: 마이그레이션 스크립트 실행 필요

【승인 확인】
이 방식으로 진행하겠습니까?
변경할 부분이 있으면 말씀해주세요.
```

---

## 4️⃣ 사이드 이펙트 분석

**다음을 모두 검토:**

```
□ 직접 영향 파일: [auth.py, routes.py, models.py]
□ 간접 영향 파일: [tests/*, client/*, docs/*]
□ 데이터베이스: [users 테이블 수정]
□ API 변경: [POST /auth/login 추가]
□ 기존 테스트: [5개 수정 필요]
□ 설정 파일: [.env JWT_SECRET 추가]
□ 배포 과정: [마이그레이션 필수]
□ 문서화: [CLAUDE.md 또는 API docs 업데이트]
```

```

  ↓
사용자: "좋습니다" 또는 "다르게 해주세요"
  ↓
GO 신호까지 구현 금지
```

## 6️⃣ 계획 수립 (복잡하면)

**3+ 단계 작업 = EnterPlanMode 필수:**

```
단계 1. auth.py 작성
  ├─ 체크: JWT 함수 완성?
Step 2. models.py 수정
  ├─ 체크: User 스키마 OK?
Step 3. routes.py 수정
  ├─ 체크: /login 작동?
Step 4. tests 수정
  ├─ 체크: 모든 테스트 통과?
Step 5. 마이그레이션
  ├─ 체크: DB 반영됨?
  ↓
사용자 승인 후 구현 시작
```

---

## 7️⃣ 구현 & 검증

```
TodoWrite로 진행상황 추적
  ├─ [ ] auth.py 작성
  ├─ [ ] models.py 수정
  ├─ [ ] routes.py 수정
  ├─ [ ] tests 수정
  └─ [ ] 마이그레이션 작성

매 단계마다:
  1. 코드 수정
  2. git diff 확인 (범위 내?)
  3. 테스트 실행
  4. 결과 보고
```

---

## 8️⃣ 완료 & 기록

```
작업 완료
  ↓
변경 내용 정리:
【구현 완료】
- auth.py: JWT 토큰 생성/검증 함수
- routes.py: POST /login, POST /logout 엔드포인트
- models.py: User 스키마 추가
- tests: 5개 테스트 수정 및 통과

【테스트】
- 모든 기존 테스트 통과
- 새로운 auth 테스트 추가 (3개)

【배포】
- 마이그레이션: python manage.py migrate
- .env: JWT_SECRET 설정 필수

【다음 단계】
(필요하면 계속)

  ↓
메모리 저장:
feedback_jwt_auth_implementation.md
  - JWT 선택 이유
  - 구현 시 주의사항
  - 테스트 커버리지
```

## ⚠️ 프로세스를 벗어나는 경우

### Case 1: 설계 제시 후 다른 방식 요청

```
사용자: "아니, JWT 말고 Session으로 해줄 수 있나?"
  ↓
Claude: "알겠습니다. Session 기반으로 재설계하겠습니다.

【새 설계】
- 방식: Session 기반
- 저장: 서버 메모리/Redis
- ...

이 방식이 맞습니까?"
  ↓
다시 승인 대기
```

### Case 2: 구현 중 설계 범위를 벗어나는 상황

```
계획: "로그인 기능만"
구현 중: "오, 비밀번호 찾기도 추가해야겠다"

  ↓
멈추고 물어보기:
"로그인 기능을 구현하다가 비밀번호 찾기도 
필요해 보이는데, 지금 추가하면 좋을까요?
아니면 따로 요청할 때 추가할까요?"
```

### Case 3: 사이드 이펙트가 예상보다 큼

```
계획: "routes.py 수정만"
실제: "테스트 10개, 문서 3개, 마이그레이션도 필요함"

  ↓
멈추고 보고하기:
"계획한 것보다 영향받는 파일이 훨씬 많습니다.
정확히 [파일들]을 수정해야 하는데, 이 정도면 괜찮으신가요?"
```

---

## ✅ 이 프로세스가 잘 작동하는 신호

- ✅ 처음부터 올바른 방향으로 구현
- ✅ 설계를 벗어나는 변경 없음
- ✅ 사이드 이펙트가 예상 범위 내
- ✅ 테스트가 첫 번째 시도에 통과
- ✅ 코드 리뷰에서 "예상과 다른 부분" 없음
