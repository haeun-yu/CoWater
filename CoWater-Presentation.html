CoWater

AI-DRIVEN AUTONOMOUS MARINE VESSEL MANAGEMENT

다중 에이전트 아키텍처 기반의
해양 무인체 통합 운영 플랫폼

CoWater란?

  • 사용자 요청을 해석하여 여러 실행안(Proposal) 생성
  • 승인된 안만 실제 임무(Mission)로 전환
  • 임무는 여러 작업으로 분해되어 Device에 할당
  • 모든 실행과 판단의 근거를 Event로 기록
  • 점진적 자동화 - 정책에 따라 자동 대응 수준 조절

현재의 과제

파편화

각 무인체마다 별도 제어 체계

추적 불가

의사결정 근거 기록 부재

경직된 자동화

수동 vs 자동, 중간 선택지 없음

취약한 대응

통신 단절 시 체계적 대응 부재

CoWater의 솔루션

5가지 핵심 철학으로 설계된 시스템

  • Action Abstraction - 하드웨어 독립적 제어
  • Planning/Execution 분리 - 계획과 실행의 명확한 구분
  • Agent 협업 - 분산형 의사결정으로 자동 조율
  • Event 추적 - 모든 결정의 근거 기록
  • 물리적 제약 극복 - 통신 환경 자동 대응

핵심 개념 1

Proposal-Mission 분리

$ PROPOSAL

시스템이 제시하는 여러 실행안. 사용자 선택 또는 정책 자동화

$ MISSION

승인된 안을 기반으로 실제 실행. Task로 분해되어 Device 할당

→ 자동화 수준을 정책으로 동적 조절 가능

핵심 개념 2

Event 기반 추적성

모든 의사결정이 Event로 기록되고 추적됩니다

  • Device Healthcheck - 배터리, 위치, 센서 상태
  • Task Dispatch - 어느 Device에 할당했는가
  • Policy Decision - 어떤 자동 결정을 내렸는가
  • Anomaly Detection - 비정상 상황 감지 및 대응
  • Mission Progress - 모든 단계의 실행 흐름

핵심 개념 3

AI Agent 기반 자동화

> 6 SYSTEM AGENTS + DEVICE AGENTS

  • RequestHandler - 요청 해석 및 경로 결정
  • MissionPlanner - Task 설계 및 Mission 생성
  • PolicyManager - 정책 기반 자동 대응
  • SystemSentinel - 실시간 이상 감시
  • DeviceBridge - 물리 장비 통신 관리
  • Device Agents - 개별 무인체 제어 및 판단

시스템 아키텍처

├─ RequestHandler
│ └─ Proposal 생성
│
├─ 사용자 승인 또는 정책 자동화

Registry Server: 중앙 저장소 | Moth: 실시간 스트림

System Agent 상세

RequestHandler

자연어 요청 해석, 경로 결정

MissionPlanner

Task 설계, Mission 생성

DeviceBridge

물리 장비 통신, 상태 동기화

PolicyManager

정책/규칙 관리, 자동 대응

SystemSentinel

이상 감시, Alert 생성

InsightReporter

분석, 통계, 리포트

Device Agent와 협력

  • 각 무인체마다 1개 Device Agent 배치
  • 자신의 Device만 직접 제어 - 분산형 제어
  • Task 수행 가능 여부 최종 판단
  • AgentConnection - Device 간 협력 관계 정의
  • 물리 통신 선택 - 유선 → RF → 음향
  • 환경 적응 - 수중 진입 시 음향 자동 전환

로컬 안전 행동: 통신 단절 시에도 로컬 정책에 따라 자율 동작
