CoWater

AI-DRIVEN AUTONOMOUS MARINE VESSEL MANAGEMENT

다중 에이전트 아키텍처 기반의
해양 무인체 통합 운영 플랫폼

완전한 추적성과 점진적 자동화를 통해
안전하고 효율적인 해양 운영 실현

CoWater란?

  • 사용자 요청을 AI가 해석하여 여러 실행안(Proposal) 제시
  • 사용자 승인
  • 승인된 안을 실제 임무(Mission)로 변환 후 실행
  • 임무는 여러 작업(Task)으로 분해되어 최적 Device에 할당
  • 모든 의사결정의 근거를 Event로 불변 기록
  • 리더 무인체가 팔로워 무인체와 협력하여 복잡한 임무 수행

핵심: 계획(Proposal)과 실행(Mission)을 분리하여 자동화 수준을 정책으로 유연하게 조절

현재 상황 vs 과제

영역 현재 상황 CoWater
운영 체계 무인체별 독립적 제어 통합 플랫폼 (통일된 인터페이스)
의사결정 근거 기록 부재 모든 결정 Event로 추적 가능
자동화 수동 운영 또는 완전 자동화만 가능 3단계 자동화 (수동 → 정책 → 자율)
협력 작업 복잡하고 수동 조율 필요 자동 협력 (AgentConnection)
통신 대응 단절 시 운영 중단 환경 자동 인식, 통신 매체 전환
확장성 무인체 추가 시 별도 구축 필요 플러그 앤 플레이 (자동 등록)

5가지 핵심 철학

① Action Abstraction

하드웨어 종속적 명령 대신 추상화된 행동을 사용해 어떤 무인체든 제어 가능

② Planning/Execution 분리

Proposal(계획)과 Mission(실행)을 분리해 자동화 수준을 유연하게 조절

③ Agent 협업

AgentConnection으로 무인체 간 동적 협력 관계 정의 (릴레이, 코디네이션)

④ Event 추적성

모든 의사결정을 불변 Event로 기록해 감사와 AI 학습 가능

⑤ 물리적 제약 극복

통신 단절, 대역폭 제한을 소프트웨어로 보상 (매체 자동 선택)

+ 분산형 의사결정

중앙 통제 없이 각 Agent이 책임 범위 내에서 자동 판단

핵심 개념 1: Proposal-Mission 분리

사용자 요청
↓
AI가 여러 Proposal(실행안) 생성
↓
사용자가 Proposal 선택 / 정책이 자동 선택
↓
선택된 Proposal → Mission으로 변환 (이 순간부터 실행 불가역)
↓
MissionPlanner가 Task로 분해
↓
각 Device의 Device Agent가 수행 가능 여부 최종 판단
↓
실행 & 모든 단계를 Event로 기록

장점 1: 투명성

계획 단계에서 여러 선택지를 제시해 사용자가 최종 결정

장점 2: 유연한 자동화

정책 변경으로 자동화 수준 동적 조절 (코드 수정 불필요)

핵심 개념 2: Event 기반 추적성

모든 의사결정이 불변 Event로 기록되어 감사, 분석, AI 학습의 기반 제공

Device Events

배터리, 위치, 센서 상태, Healthcheck

Task Events

할당, 수락, 시작, 진행, 완료, 실패

System Events

Proposal 생성, Approval, 정책 판단, 이상 징후

분석 사례

  • ✓ 감사: "왜 이 Task를 A Device에 할당했나?" → Event에서 즉시 추적
  • ✓ 학습: 성공/실패 Task의 Event 패턴 분석 → AI 모델 개선
  • ✓ 병목: 각 단계별 소요 시간 분석 → 운영 최적화

6개 System Agent 상세

RequestHandler

책임: 자연어 사용자 요청 해석, 어떤 Agent에 위임할지 경로 결정

입력: 자연어 명령

출력: Proposal 또는 직접 응답

MissionPlanner

책임: Proposal 기반으로 Mission 생성, Task 분해, 생명주기 관리

입력: 승인된 Proposal

출력: Mission, Task 할당

DeviceBridge

책임: 물리 무인체와 양방향 통신, 상태 동기화, 신호 정규화

입력: Device로부터 상태, Heartbeat, Problem Report

출력: Task 전달, Event 발행

PolicyManager

책임: 정책/규칙 관리, 자동 대응 결정, 기기 생명주기 관리

입력: Event, System 상태

출력: 자동 결정, 정책 실행

SystemSentinel

책임: 지속적 이상 징후 감시, Alert 생성, 건전성 체크

입력: Healthcheck, Event

출력: Alert, SYS_ANOMALY_DETECTED Event

InsightReporter

책임: 데이터 조회, 통계 분석, 리포트 생성

입력: Registry의 모든 데이터 (읽기 전용)

출력: Report, Insight, 통계

Device Agent와 협력

계층적 구조

Middle-Layer Agent (Control Ship) → 릴레이
↓
Lower-Layer Agents (USV, AUV, ROV)
↓
각 Aagent가 AgentConnection 기반으로 협력

  • 독립성: 각 Device Agent는 자신의 Device만 직접 제어
  • 최종 판단: Task 수신 후 자신의 Capability/Resource/Safety 확인해 수락/거절
  • 협력: AgentConnection으로 물리 통신 매체(유선/RF/음향) 기반 협력 정의
  • 환경 적응: AUV가 수중 진입 → 자동으로 음향 통신으로 전환
  • 로컬 안전 행동: 통신 단절 시에도 로컬 정책에 따라 자율 동작

Communication Media Priority

1순위: Wired (테더) → 2순위: RF/Internet → 3순위: Acoustic (음향)
환경(수상/수중) 자동 감지로 최적 매체 선택

성공 시나리오

시나리오 1: 항만 기뢰 탐지

명령: "항만 주변 300m 기뢰 탐지"

CoWater:

  1. Proposal 3가지 생성 (경로, 깊이, 속도 조합)
  2. 사용자가 안전한 옵션 선택
  3. AUV에 Task 할당 → 자동 실행
  4. 모든 단계 Event로 기록

시나리오 2: 통신 단절 대응

상황: AUV가 수중 진입 → RF 신호 손실

CoWater:

  1. 환경 변화 감지 → Event 발행
  2. AgentConnection 재구성 (음향)
  3. AUV가 로컬 정책으로 운영
  4. 신호 복구 후 자동 재동기화

시나리오 3: 정책 기반 자동화

정책: "배터리 20% 이하 → 자동 귀항"

CoWater:

  1. Healthcheck에서 배터리 20% 감지
  2. Policy 규칙 자동 실행
  3. 귀항 Mission 자동 생성
  4. 사용자 알림 + Event 기록

시나리오 4: 복합 임무

명령: "USV와 AUV가 협력해 수심 측정"

CoWater:

  1. Proposal에서 협력 방식 제시
  2. AgentConnection으로 릴레이 설정
  3. 각 Device가 역할 수행
  4. 데이터 자동 수집 & 동기화
