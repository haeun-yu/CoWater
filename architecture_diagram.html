🏗️ CoWater 시스템 구성
4개의 Modular Docker Compose 파일로 구성
각 파일은 특정 목적에 맞게 독립적으로 실행 가능하며, 환경변수로 연결됩니다.
📦 docker-compose.core.yml
✅ 핵심 인프라 + 최소 에이전트
목적: 기본 시스템 운영 (데이터 수집 + 기본 API)
용도: 개발/테스트, 핵심 기능만 필요한 경우
# docker-compose.core.yml 포함 서비스 📊 데이터 계층 └─ postgres (TimescaleDB + PostGIS) └─ redis (Pub/Sub 버스) 🔌 API & 게이트웨이 └─ core (REST API + WebSocket 허브) └─ moth-bridge (Moth/RSSP 수신) └─ agent-gateway (Nginx 프록시 - 모든 에이전트 관리) ⚙️ 에이전트 └─ control-agents (Chat 에이전트) └─ supervision-agents (시스템 모니터링) 🎨 UI └─ frontend (Next.js 대시보드) # 실행 docker compose -f docker-compose.core.yml up -d
특징: postgres, redis 모두 포함 (standalone 가능) • 모든 UPSTREAM은 기본값 (localhost:8001)
📦 docker-compose.agents.yml
🔌 모든 에이전트 + 의존 인프라
목적: 전체 파이프라인 (탐지→분석→응답→리포팅)
용도: 프로덕션, 모든 기능 필요
# docker-compose.agents.yml 포함 서비스 (docker-compose.core.yml의 모든 서비스 + 추가) ⚙️ 추가 에이전트 └─ detection-agents (Rule 기반 탐지) └─ analysis-agents (AI 기반 분석) └─ response-agents (경보 생성) └─ report-agents (AI 리포트 생성) └─ learning-agents (피드백 기반 학습) # 실행 docker compose -f docker-compose.agents.yml up -d # 특정 서비스만 실행 가능 docker compose -f docker-compose.agents.yml up -d detection-agents analysis-agents
특징: docker-compose.core.yml의 상위집합 • 모든 에이전트 포함 • report-agents가 사용할 DB 포함
📦 docker-compose.simulator.yml
🎮 시뮬레이터 (선택)
목적: YAML 시나리오 기반 AIS 데이터 생성
용도: 테스트, 데모, 훈련
# docker-compose.simulator.yml 포함 서비스 🎮 시뮬레이션 └─ redis (Pub/Sub 버스만 필요) └─ simulator (YAML 시나리오 기반 데이터 생성) # 실행 (다른 서비스는 별도로 실행) docker compose -f docker-compose.core.yml up -d # 먼저 코어 실행 docker compose -f docker-compose.simulator.yml up -d # 시뮬레이터 실행 # 특정 시나리오로 실행 SCENARIO=collision_risk docker compose -f docker-compose.simulator.yml up -d
특징: redis만 포함 • 다른 compose 파일과 함께 사용 • 실제 Moth 서버 대신 동작
📦 docker-compose.yml (기본)
⭐ 기본 구성 (docker-compose.agents.yml과 동일)
목적: 기본 docker compose 명령어로 모든 서비스 실행
용도: 대부분의 경우 이 파일 사용
# docker-compose.yml 사용 cd infra docker compose up -d # 모든 서비스 실행 docker compose down # 모든 서비스 중지 docker compose logs -f core # 로그 확인
내용: docker-compose.agents.yml과 동일 (참고: 실제 내용 확인 필요)

🔧 컨테이너 실행 순서
1️⃣ postgres 시작 ↓ (healthcheck: pg_isready) 2️⃣ redis 시작 ↓ (healthcheck: redis-cli ping) 3️⃣ core 시작 ├─ 의존성: postgres (healthy), redis (healthy) ↓ 4️⃣ moth-bridge 시작 ├─ 의존성: redis (healthy) ├─ → platform.report.* Redis 발행 시작 ↓ 5️⃣ control-agents 시작 ├─ 의존성: redis (healthy), core (started) ↓ 6️⃣ agent-gateway 시작 ├─ 의존성: control-agents (started) ├─ → :7701-7709 리스닝 시작 ↓ 7️⃣ detection-agents, analysis-agents, ... 시작 ├─ 의존성: redis (healthy), core (started) ├─ → Redis pub/sub 구독 시작 ↓ 8️⃣ frontend 시작 ├─ 의존성: core (started) └─ → http://localhost:7702 접근 가능

📊 각 단계의 데이터 형식
단계 데이터 소스 처리 방식 출력 데이터
1. 수집 Moth/RSSP WebSocket 정규화 + DB 저장 platform_reports (위치, 항속, 침로, 상태)
2. 탐지 platform_reports 규칙 엔진 (CPA, Anomaly 등) detection_results (alert_type, confidence)
3. 분석 detection_results LLM 프롬프팅 analysis_results (reasoning, assessment)
4. 응답 analysis_results 심각도 결정 alerts (severity: critical/warning/info)
5. 리포팅 alerts + platform_reports LLM 문서 생성 reports (마크다운 본문)
6. 학습 alerts + user_feedback 통계 분석 learning_parameters (임계값 제안)

채널 발행자 구독자 데이터 포맷 설명
platform.report.{platform_id} moth-bridge core, detection-agents {"mmsi": int, "lat": float, "lon": float, "sog": float, ...} 선박 위치 + 상태 (Moth에서 파싱)
detect.{agent_id} detection-agents analysis-agents {"alert_type": "cpa|anomaly|zone|distress", "confidence": float} 탐지 결과 (규칙 기반)
analyze.{agent_id} analysis-agents response-agents {"assessment": str, "reasoning": str, "risk_level": float} 분석 결과 (AI 기반)
respond.{severity} response-agents core, report-agents, learning-agents {"severity": "critical|warning|info", "message": str} 경보 (심각도별 브로드캐스트)
report.{report_id} report-agents core {"report_type": "summary|detailed|incident", "content": str} AI 생성 리포트
learn.rule_update.{agent_id} learning-agents detection, analysis, response {"parameter": str, "new_value": any, "reason": str} 규칙 개선 제안
user.{event_type} frontend supervision, learning {"event": str, "user_id": str, "timestamp": str} 사용자 상호작용
system.alert_acknowledge frontend learning {"alert_id": uuid, "acknowledged_by": str} 경보 승인
platform:state:{platform_id} moth-bridge (캐시) Key/Value 캐시 (TTL: 60초) 빠른 조회용 상태 캐시

🔧 문제 해결
문제 원인 해결책
detection-agents가 Redis 연결 실패 REDIS_URL 주소 오류 또는 REDIS_BIND_ADDR≠0.0.0.0 호스트 A에서 REDIS_BIND_ADDR=0.0.0.0 확인 + 방화벽 규칙 확인
platform.report.* 메시지 안 나옴 moth-bridge가 MOTH_SERVER_URL에 연결 안됨 호스트 A에서 docker logs cowater-moth-bridge 확인
agent-gateway가 detection 못 찾음 DETECTION_AGENTS_UPSTREAM 주소 오류 호스트 A: docker logs cowater-agent-gateway 확인, 호스트 B IP 정확한지 확인
Port already in use 같은 포트에 다른 컨테이너 실행 중 docker compose down 후 재실행 또는 BIND_ADDR 변경
