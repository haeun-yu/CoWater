# 예외 및 자동 대응 (Exceptions & Auto-Recovery)

오류 상황과 시스템의 자동 대응 메커니즘

## 주요 예외 시나리오

### 1. LLM 경로 검증 실패
**상황**: Agent가 지정된 LLM 경로에 접근 불가

**자동 대응**:
1. 재시도 (exponential backoff)
2. 대체 LLM 경로로 폴백
3. 관리자 알림

**복구**: 경로 설정 검증 후 재시도

### 2. 미션 오프라인
**상황**: 해당 선박/자산이 네트워크 연결 손실

**자동 대응**:
1. 상태를 "Offline" 으로 표시
2. 새 작업 제안 중단
3. 재연결 대기

**복구**: 네트워크 복구 시 자동 재활성화

### 3. 작업 실행 실패
**상황**: Operation 실행 중 오류 발생

**자동 대응**:
1. 상태를 "Failed" 로 기록
2. 에러 로그 저장
3. 재시도 정책 적용

**복구**: 관리자 검토 후 수동 재시도 또는 대체 작업 제안

## 크리티컬 자동 대응

| 조건 | 자동 동작 | 알림 |
|------|----------|------|
| 3회 연속 실패 | 미션 일시 정지 | 관리자 이메일 |
| LLM 할당량 초과 | 새 제안 중단 | 대시보드 경고 |
| 데이터 무결성 오류 | 트랜잭션 롤백 | 개발팀 알림 |
