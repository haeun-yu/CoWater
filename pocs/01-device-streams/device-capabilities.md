# 01 시뮬레이터 디바이스 기능 목록

이 문서는 `pocs/01-device-streams` 시뮬레이터 기준으로 각 디바이스가 **현재 할 수 있는 기능**과, 실제 운영용으로 **추가 구현이 필요한 기능**을 정리한 목록입니다.

- 기능명은 코드나 API에서 쓸 수 있게 `snake_case` 형태로 적었습니다.
- `현재 가능`은 지금 시뮬레이터가 이미 데이터 생성 또는 이동을 흉내 내는 기능입니다.
- `추가 필요`는 시뮬레이터나 제어 API를 더 붙여야 실제로 동작하게 만들 수 있는 기능입니다.

## 공통 기능

| 기능명 | 상태 | 한국어 설명 |
|---|---|---|
| `position_update` | 현재 가능 | 현재 위치 위경도와 고도를 주기적으로 갱신합니다. |
| `heading_update` | 현재 가능 | 진행 방향(헤딩)을 계속 바꿉니다. |
| `speed_update` | 현재 가능 | 이동 속도를 범위 안에서 계속 바꿉니다. |
| `telemetry_report` | 현재 가능 | 위치, 자세, 센서 값을 묶은 텔레메트리 데이터를 전송합니다. |
| `sensor_stream` | 현재 가능 | 센서별 데이터를 개별 트랙으로 전송합니다. |
| `route_move` | 추가 필요 | 여러 지점을 순서대로 이동하는 경로 주행 기능입니다. |
| `move_to_device` | 추가 필요 | 특정 디바이스 위치를 목표로 이동하는 기능입니다. |
| `patrol_route` | 추가 필요 | 정해진 경로를 반복 순찰하는 기능입니다. |
| `follow_target` | 추가 필요 | 특정 디바이스나 좌표를 추적하며 따라가는 기능입니다. |
| `return_to_base` | 추가 필요 | 시작 위치나 기준 거점으로 복귀하는 기능입니다. |
| `emergency_stop` | 추가 필요 | 즉시 이동과 작업을 멈추는 비상 정지 기능입니다. |

## 요약

- 현재 구현은 주로 `위치 이동 + 센서 스트림 전송` 중심입니다.
- 실제 임무형 동작은 `patrol_route`, `move_to_device`, `follow_target`, `charge_at_tower`, `light_on/off` 같은 액션을 추가해야 합니다.
- `USV`는 수면 이동, `AUV`는 수중 이동과 수심 제어, `ROV`는 수심이 큰 정밀 작업에 가장 잘 맞습니다.
